Kangle 做HTTP匿名代理

Date 五 17 五月 2013 By eleven.i386 Category Linux运维. Tags 代理 翻墙

kangle web服务器（简称：kangle）是一款跨平台、功能强大、安全稳定、 易操作的高性能web服务器和反向代理服务器软件。除此：kangle也是一款专为做虚拟主机 研发的web服务器。实现虚拟主机独立进程、独立身份运行。用户之间安全隔离，一个用户 出问题不影响其他用户。安全支持php、asp、asp·net、java、ruby等多种动态开发语言。 百度百科

安装

下载源码 wget http://www.kanglesoft.com/download/src/kangle-3.2.3.tar.gz

编译环境

apt-get install build-essential

安装pcre库

apt-get install libpcre++-dev

安装ssl库

apt-get install libssl-dev

编译

./configure –prefix=./kangel
make 
make install

配置

安装完之后,在./kangle/etc/下有一个config.xml 文件

可以在这里修改 http监听端口 和web管理端口, 以及其他的选项, 当然也可以直接启动kangle然后通过web管理界面进行选项修改

设置匿名代理

在web控制台–请求控制–插入一条新的规则

在可用的标记模块中选择flag

然后勾选no_x_forared_for,点击提交

结语

设置本机浏览器的代理地址为kangle地址之后 访问www.ip138.com会发现客户端的ip变成了代理服务器的ip, 实现了匿名访问

其实做这种正向代理, 用nginx 也是完全可以的.没必要用这种不是很信任的软件
