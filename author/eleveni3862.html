记一次安装fcitx过程
一 11 三月 2013 by eleven.i386

今天实在无法忍受ibus的启动速度和词汇准确性了. 将ibus更换成fcitx, 

apt-get install fcitx
im-switch -s fcitx
im-switch -s fcitx -z default

然后在autostart中加入fcitx的自启动.. 注销X界面之后, 发现fcitx是启动起来了, 只是应用程序的默认输入法还是Ibus, 于是google之 ,找到了fcitx环境变量设置, 在~/.xprofile 中加入fcitx的环境变量:

export GTK_IM_MODULE=fcitx
export XMODIFIERS=@im=fcitx
export QT_IM_MODULE=fcitx

检查

记. 一次邮箱密码丢失
一 17 九月 2012 by eleven.i386

话说某年某月某天, 某人丢失公司分配的邮箱密码,

在虚拟机中的windows,用foxmail客户端, 因为某些原因密码丢失,而公司邮件又需要用 该客户端发送,于是就有了下面的过程,

在物理机中的雷鸟客户端保存了密码, 首先需要客户端上的密码加密,主要是smtp密码传输加密

好了,然后就用tcpdump 就可以了,

tcpdump -X -i eth0 port 587 -vv

TLS/STARTTLS 端口是587 ,雷鸟和foxmail都自动认出来端口是587,所以我也捕抓发往587端口的数据

从结果中分析,下图是客户端跟服务端验证过程

得到一个base64的密码,

echo -n "password" | base64 -d

解开就是明文密码

记.遗忘RTX密码
日 22 九月 2013 by eleven.i386

引子

在虚拟机中rtx密码是保存的,第一次拿到密码,没有做第三方的记录. 中秋期间,wine了一个rtx. 结果发现自己忘记密码了

过程

由于有了之前的遗忘密码事件 (遗忘密码之svn,记.一次邮箱密码丢失)

一开始的思路是在登录rtx的时候抓包分析,rtx的验证数据包, 不过还是习惯性的google了一下 没想到真找到了更简单的方法.

有资料显示rtx在内存中明文存储密码 by the goole

WINHEX打开RTX的内存

  1. winhex打开rtx的内存

    1. 点击图片中1的位置.open Ram
    2. 在窗口中找到rtx.
    3. 选择主要内存.Primary Memory

  2. 查找16进制字符串40000000500061007300730077006F0072006400

    1. 点击图片中1的位置.Find Hex values
    2. 在文本框内输入要搜索的16进制字符串40000000500061007300730077006F0072006400

  3. password下面的位置就是明文密码

结果

好了,明文密码得到了.只是这样的rtx是否安全呢...

加密你的HTTP请求(穿墙)
五 17 五月 2013 by eleven.i386

之前用kangle在香港的vps上做了一个http代理 但是在使用的时候发现,访问国外的一些站点 还是无法正常访问,但是在vps上面访问正常, 而且kangle日志里面看见国外那些站点返回的状态码也是200,说明vps请求那些站点是没问题的 那么问题就是发生在客户端跟vps之间了,因为这一段的数据是明文的HTTP请求, 很容易被上层运营商过滤掉 那么我们就将客户端跟vps之间这段http过程加密就好了,

Stunnel

Stunnel 是一种程序，使程序员和系统管理员可以很轻松地对任意 TCP 会话加密。 您可以很轻松地在客户机和服务器上启用 SSL ― 而且这样做不会影响程序源代码。 Stunnel 服务器主要执行两个功能： 一，首先，接收未加密的数据流，进行 SSL 加密，然后将其通过网络发送； 二，对已进行 SSL 加密的数据流进行解密，并将其通过网络发送给另一个程序 （该程序通常驻留在同一机器上，以避免本地网络上的窥探攻击）。
